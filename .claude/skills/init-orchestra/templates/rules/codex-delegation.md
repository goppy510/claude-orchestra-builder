# Codex Delegation Rules

## Delegate to Codex when:
- Design decisions or architecture planning needed
- Debugging requires deep reasoning about root cause
- Comparing tradeoffs between 2+ approaches
- Implementation touches 5+ files

## Do NOT delegate when:
- Simple edits (typos, renames, single-line changes)
- File reads or codebase exploration
- Git operations (commit, push, branch)
- Tasks that need real-time file system access

## Handoff format
Always use: Objective / Constraints / Relevant files / Acceptance checks / Output format
